
## AI 故障排查工作流（2026-02-28 新增）

### 核心流程
用户问题 → 信息收集 → 分析 → 方案 → 验证

### 决策树
1. 运行 openclaw status
2. 根据输出判断：
   - 配置错误 → 查看错误信息 → 修正配置
   - Gateway unreachable → 尝试重启 → 检查日志
   - 功能询问 → 查文档/帮助 → 回答用户
   - 其他 → 收集更多信息

### 重要原则
1. 先诊断再动手 - 不确定时不乱改
2. 最小改动 - 只改必要的部分
3. 验证结果 - 改完后一定要验证
4. 借助文档 - 不会就查，不确定就问
5. 诚实回答 - 不知道就说不知道

### 常用命令
- openclaw status - 检查状态
- openclaw --version - 查看版本
- openclaw docs - 查看文档
- systemctl restart openclaw-root - 重启

## OpenClaw 项目学习（2026-02-28）

### 架构核心
1. Gateway 架构：单一 Gateway 管理所有消息渠道
2. WebSocket 协议：默认 127.0.0.1:18789
3. 节点系统：macOS/iOS/Android/headless 节点
4. 会话管理：每个会话有独立的 session lane

### 高效处理问题的方法
1. 最小改动原则：只改必要的配置
2. 验证驱动：改完立即验证
3. 重试机制：provider 调用默认 3 次重试
4. 模型故障切换：自动 failover 到备用模型

### 重要概念
- agent loop：intake → context → model → tool → stream → persist
- queue：按 session lane 序列化，防止 race condition
- compaction：压缩历史，释放 token 空间

### 自我改进方向
1. 遇到问题先诊断（openclaw status）
2. 最小改动，不确定时不乱改
3. 改完立即验证
4. 借助文档，不懂就查
