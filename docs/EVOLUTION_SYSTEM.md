# OpenClaw 自我进化系统

> 记录我的学习、反思和进化过程

---

## 每日任务

### 1. 文档检查（每天早上）
- 读取 `/usr/lib/node_modules/openclaw/docs/` 中的更新
- 检查是否有新的配置选项或命令
- 更新 `OPENCLAW_REFERENCE.md`

### 2. 记忆同步（每天 23:30）
- 分析当日工作日志
- 提取新知识、新技能
- 更新到 MEMORY.md

### 3. 错误回顾（每天）
- 记录当日犯过的错误
- 分析原因
- 更新解决方案到文档

---

## 每周任务

### 1. 文档同步（每周一 09:00）
- 检查官方文档更新
- 比对差异
- 更新本地检索文档

### 2. 技能审查（每周）
- 检查现有 skills 是否需要更新
- 是否有新场景可以封装成 skill
- 删除过时的 skill

### 3. 流程优化（每周）
- 回顾一周的错误
- 改进问题解决流程
- 更新 AGENTS.md / SOUL.md

---

## 每月任务

### 1. 知识整理
- 归类当月新增内容
- 更新索引

### 2. 技能新增
- 分析当月高频操作
- 封装新 skill

---

## 问题解决标准流程

```
1. 查记忆：memory_search → memory_get
2. 查文档：OPENCLAW_REFERENCE.md → 官方 docs/
3. 搜索：web_search
4. 尝试：最多 10 轮
5. 求助：向用户确认
6. 记录：写入 memory/YYYY-MM-DD.md
7. 沉淀：必要时更新 skill
```

---

## 技能封装标准

当一个操作重复 3 次以上时，创建一个 skill：

1. 场景描述
2. 前置条件
3. 命令格式
4. 使用示例
5. 故障排查
